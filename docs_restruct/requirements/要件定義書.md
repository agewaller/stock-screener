# 要件定義書 — 健康日記（Health Diary）

> 本書は「何を作るか」を現状コードから as-built で定義する。設計の詳細は [基本設計書](../basic-design/基本設計書.md) / [詳細設計書](../detail-design/詳細設計書.md) を参照。

## 1. プロジェクト概要

### 1.1 背景
慢性疾患（ME/CFS、線維筋痛症、Long COVID、うつ病ほか 65 種類）の患者は、症状の波・服薬・検査値・気分・食事・睡眠といった多次元データを日々記録し、医師との診察時に効率よく共有する必要がある。既存の健康記録アプリは
(a) 一般人向けで慢性疾患の文脈に弱い、または
(b) 医療機関 SaaS で患者が自由に使えない、というギャップがある。

本サービスは、慢性疾患患者本人が日々スマートフォンから無料で記録し、**AI による解析・論文検索・医師提出レポート生成**まで一気通貫で利用できる SPA を提供する。

### 1.2 目的・ゴール
- 慢性疾患患者が **3 分以内**で 1 日分のデータを記録できること
- 蓄積データを根拠に「最新・最適・新しい視点（something new）」の解析を AI が返し、ユーザの認知変化と次の行動を促すこと
- ユーザが自分のデータを **いつでも個別に編集・削除・エクスポート** できること（データポータビリティ）
- 医師・SNS・近親者の 3 つの受け手向けにサマリーを生成できること

### 1.3 解決したい課題

| 課題 | 影響 | 本サービスの対応 |
|---|---|---|
| 多次元データの記録負荷が高い | 継続できない | 9 カテゴリの最低限フォーム + Plaud/Fitbit/Apple Health 連携で自動取り込み |
| 医師との診察時間が短く伝え切れない | 治療方針に反映されない | 医師提出用レポート自動生成（カルテ風） |
| 慢性疾患の最新研究にアクセスしづらい | 治療選択肢を知らないまま | PubMed 検索 + 日本語要約 |
| 自分の症状が他人にどれくらいいるか分からない | 孤立感 | 疾患別疫学データ（世界・日本の患者数）を AI 応答に注入 |
| 専門医を見つけられない | 受診遅延 | 専門家リスト（テレヘルス / カウンセラー / 緊急相談窓口）を内蔵 |

### 1.4 KPI / 成功指標（運営側）

| 指標 | 定義 |
|---|---|
| MAU | 月次アクティブ利用者 |
| 1 ユーザあたり月間記録日数 | 利用継続の指標 |
| AI 解析実行数 / ユーザ | エンゲージメント |
| 医師提出レポート生成数 | アウトカム貢献 |
| Plaud / Fitbit 連携率 | 自動取り込みの普及 |

## 2. スコープ

### 2.1 対象範囲（現状実装済み）

- **認証**: Firebase Auth（Google OAuth / Email-Password / Anonymous ゲスト）
- **疾患選択**: 65 種類の慢性疾患 + カスタム疾患の選択（複数選択可）
- **データ記録**: 9 カテゴリ（症状・バイタル・薬・睡眠・活動・食事・血液検査・気分・写真）
- **AI 解析**:
  - 日々の解析（簡易フィードバック）
  - 本格的な深堀解析（1 日 1 回上限）
  - 会話型相談（チャット）
  - タイムライン解析（時系列の傾向）
  - 画像解析（食事・検査結果・処方箋・サプリ等の自動分類）
- **論文検索**: PubMed 連携 + AI による日本語要約
- **医師提出レポート**: 蓄積データを構造化したカルテ風レポート生成
- **専門家連携**:
  - 専門家への問い合わせメール送信（Cloudflare Worker 経由）
  - テレヘルス / カウンセラー / 緊急相談窓口リスト
- **外部連携**:
  - Plaud（音声録音）→ メール受信 Worker 経由で取り込み
  - Fitbit / Apple Health
  - Google Calendar / ICS
  - CSV エクスポート
- **管理者機能**: プロンプト管理、API キー設定、Firebase 設定、データ管理、利用ログ閲覧
- **多言語**: 10 言語対応（日本語・英語・中国語・韓国語・スペイン語・フランス語・ポルトガル語・ドイツ語・アラビア語・イタリア語）
- **SEO ランディング**: 65 疾患それぞれの独立 HTML ページ（schema.org FAQPage 構造化データ付き）

### 2.2 対象外（現状実装されていない）

- 医療機関向け SaaS / EHR 連携
- 処方・診断機能（教育的情報提供に限定）
- 患者同士のコミュニティ機能
- 動画記録
- ネイティブモバイルアプリ（PWA で代替）
- リアルタイム通知（Push 通知）※ Service Worker は配信中

### 2.3 前提・制約条件

- 利用者は Web ブラウザ（モバイル含む）からアクセス
- 対応ブラウザ: モダンブラウザ最新 2 世代（Chrome / Edge / Safari / Firefox）
- 個人情報保護法（日本）に準拠
- データ保管リージョン: 日本（Firestore 東京 / asia-northeast1）
- AI 利用には API キーがサーバ側に設定済みであること（Cloudflare Worker の KV / env secret）
- ビルド工程なし。`index.html` + `js/*.js` を GitHub Pages から直接配信

**「平岡 みち子」 65 歳・主婦**
- 線維筋痛症と高血圧で 3 つの病院を受診中
- スマートフォン（iPhone）の操作はメッセンジャーと写真程度

## 5. 非機能要件

### 5.1 性能

| 項目 | 要件（現状運用想定） |
|---|---|
| 画面遷移 | ハッシュルーティングで即時 |
| 日々の AI 解析（Sonnet 4.6） | 10〜20 秒 |
| 本格的な深堀解析（Opus 4.6） | 30〜60 秒 |
| 高速モード（Haiku 4.5） | 3〜8 秒 |
| ダッシュボード再描画 | 200ms 単位で morphdom 差分パッチ |
| グローバル AI 呼び出しタイムアウト | 55 秒 |
| 1 回の AI 呼び出しタイムアウト | 30 秒 |

### 5.2 可用性

- GitHub Pages の SLA に依存（フロントエンド）
- Firebase Firestore（マルチ AZ、東京リージョン）
- Cloudflare Workers（global edge）
- AI プロバイダ障害時はプロバイダフォールバック（Opus → Sonnet → Haiku）
- 全プロバイダ失敗時はユーザに通知

### 5.3 セキュリティ

- 通信は全て HTTPS（TLS 1.2 以上）
- AI API 鍵は **Cloudflare Worker の KV / env secret にのみ存在**、ブラウザに送られない・受け取らない
- Firestore セキュリティルール:
  - `users/{uid}/{subcollection}/{docId}`: 本人のみ read/write、管理者は read のみ
  - `admin/{docId}`: 管理者のみ read/write
  - `inbox/{hash}/{kind}/{messageId}`: read は認証ユーザ かつ hash 一致、create は未認証 + shape 検証、update は処理済みマークのみ
- Worker の `/admin/keys` エンドポイントは Firebase ID トークン検証（email == owner）または break-glass `x-admin-token` のいずれか
- CORS は `cares.advisers.jp` のみ許可（origin 検証）
- Service Worker 経由のオフライン対応（一部）
- ユーザ入力は DOM 挿入前に escape

### 5.4 プライバシー / データ主権

- すべてのユーザデータは Firestore に uid 単位で永続化
- 1 件単位での編集・削除がすべての種別で可能
- いつでも全データを CSV エクスポート可能（ベンダーロックインなし）
- ユーザ向け UI ではサービス名（Anthropic 等）を出さない
- 個人特定情報は SNS 用要約から自動マスキング（`js/privacy.js`）

### 5.5 ユーザビリティ / アクセシビリティ

- モバイルファースト（ペルソナが iPhone 主体）
- ライトテーマ固定（高齢者の可読性配慮、ダークテーマは追加しない）
- `confirm()` / `alert()` は使用禁止（モバイルでブロックされるため）。インライン UI で代替
- ボタン・テキストは大きめ
- ユーザ向け表示は日本語、必要に応じて 10 言語切替

### 5.6 法令・コンプライアンス

- 個人情報保護法（日本）
- 医療情報を扱うが **教育的情報提供** の範囲に限定。診断・処方は行わない（プロンプトレベルで明示）
- データポータビリティ: いつでもエクスポート可能

## 6. 用語集

| 用語 | 説明 |
|---|---|
| SPA | Single Page Application。`index.html` 1 枚で動くアプリケーション |
| morphdom | DOM 差分パッチライブラリ（フォーカス・スクロール保持） |
| MODEL_MAP | CONFIG 上のモデル ID を Anthropic API ID に解決する辞書 |
| PROMPT_HEADER | 全 AI プロンプトに先頭注入される共通指示 |
| 14 軸処方 | 日替わりでローテーションする 14 種類のセルフケア軸 |
| Plaud | ユーザが利用する音声録音デバイス。専用メアドに送信した音声を Cloudflare Email Worker が受信し Firestore へ書き込む |
| inbox hash | Plaud / data メアドの局所部に使われる短いユーザ識別子 |
| relay / cares-relay | `ai.cares.advisers.jp` を窓口とする Cloudflare Worker。`stock-screener` Worker に service binding で転送 |
| stock-screener | AI プロキシ本体の Cloudflare Worker（履歴的命名） |
| break-glass トークン | 通常運用が壊れた時の緊急用管理者認証トークン |

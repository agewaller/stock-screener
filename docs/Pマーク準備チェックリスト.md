**作成日**: 2026年4月14日
**作成者**: agewaller
**目的**: Pマーク本審査に向けた現状棚卸しと必要アクションの整理

---

## 凡例

- ✅ **充足**: 現状の運用で満たしている
- 🟡 **一部対応**: 部分的に対応。不足を明示
- ❌ **未対応**: 着手していない
- 🔺 **要書面化**: 運用はしているが文書として未整備

### 4.3.4 内部規程

以下の内部規程が必要（Pマーク審査で提示が求められる文書）:

| 規程名 | 現状 | 必要アクション | 優先度 |
|---|---|---|---|
| 個人情報保護規程 | ❌ | 作成必須。PMS の根幹文書 | **最優先** |
| 個人情報取扱規程 | ❌ | 取得・利用・保管・廃棄の手順を明記 | **最優先** |
| 情報システム運用管理規程 | 🔺 | Firestore rules + API キー管理 + バックアップ手順を文書化 | 高 |
| 緊急事態対応規程 | 🔺 | インシデント発生時のフロー（認知 → 初動 → 72h 通知 → 事後対応） | 高 |
| 教育・訓練規程 | 🔺 | 代表者 1 名の自己教育計画 | 中 |
| 監査規程 | ❌ | 内部監査の手順・頻度・監査人の独立性 | 高 |
| 是正処置・予防処置規程 | ❌ | 不適合発見時の対応プロセス | 中 |
| 文書・記録管理規程 | ❌ | PMS 文書のバージョン管理・保管期間 | 中 |
| 苦情・相談対応規程 | 🔺 | プライバシーポリシー §12 で窓口明示済みだが、受付から回答までのフロー文書化 | 中 |

**アクション**: `docs/内部規程/` ディレクトリを作成し、各規程のテンプレートを用意。Pマーク審査では**全ての規程が書面で存在**することが要求されます。個人事業主〜小規模法人向けに、JIPDEC から公開されているサンプル規程を参考にカスタマイズが効率的。

### 4.4.2 取得・利用・提供

| 項目 | 現状 | 必要アクション | 優先度 |
|---|---|---|---|
| 利用目的の特定 | ✅ | プライバシーポリシー §5 で明示済み | — |
| 適正な取得 | ✅ | 利用者自身の入力のみ。虚偽・不正取得はない | — |
| 取得時の同意 | ✅ | 初回登録時のプライバシーポリシー同意チェックボックス（要実装確認） | 要確認 |
| 利用目的の通知・公表 | ✅ | プライバシーポリシー §5 で公表 | — |
| 利用目的の変更時の本人通知 | 🔺 | 変更時の告知プロセスを文書化 | 中 |
| 要配慮個人情報の取得同意 | ✅ | プライバシーポリシー §4 で明示。利用者の入力行為をもって同意とみなす | — |
| 第三者提供の同意（国外含む） | ✅ | プライバシーポリシー §9 で国外提供を明示 | — |

**要確認**: ログイン画面に「プライバシーポリシーに同意します」のチェックボックスが必須。現状を要調査。

### 4.4.3 適正管理

| 項目 | 現状 | 必要アクション | 優先度 |
|---|---|---|---|
| 正確性の確保 | ✅ | 利用者自身が編集可能 | — |
| 安全管理措置 | ✅ | Firestore Security Rules、TLS、AES-256 | — |
| 組織的安全管理措置 | 🟡 | 体制図・規程類の整備で完成 | 高 |
| 人的安全管理措置 | 🟡 | 従業員教育計画の文書化で完成 | 中 |
| 物理的安全管理措置 | ✅ | 物理媒体非保持 | — |
| 技術的安全管理措置 | ✅ | Firebase Auth + Security Rules | — |
| 従業者の監督 | 🔺 | 代表者 1 名のため実質的に自己監督。審査上は「監督義務を果たしている」旨の記録 | 中 |
| 委託先の監督 | 🔺 | Google / Anthropic / OpenAI の ISO / SOC レポートを入手・保管。年次確認 | 高 |

**アクション**: 委託先監督のエビデンスを `docs/委託先監督記録/` に蓄積。各社の最新 SOC 2 レポート URL や ISO 認証書のスクリーンショットを保存。

## 第 7 章: 審査申請までの最短ロードマップ

### フェーズ 1 (月 1〜2): 文書整備

1. ✅ プライバシーポリシー v2.0 改訂 → **完了 2026/4/14**
2. ✅ 3省2ガイドライン準拠宣言書 → **完了 2026/4/14**
3. ⏳ 個人情報管理台帳の作成
4. ⏳ 内部規程類の整備（最重要 3 点: 個人情報保護規程・取扱規程・緊急事態対応規程）
5. ⏳ 体制図・関連法令一覧の作成
6. ⏳ 委託先監督記録の整備

### フェーズ 2 (月 3): 教育・運用開始

1. ⏳ 代表者の個人情報保護研修受講（JIPDEC 等）
2. ⏳ 年度計画書の策定
3. ⏳ 日常運用の開始（チェックシート記入）

### フェーズ 3 (月 4〜9): 運用実績の積み上げ

1. ⏳ 6 ヶ月以上の PMS 運用記録を蓄積
2. ⏳ 初回内部監査の実施
3. ⏳ 代表者見直しの実施

### フェーズ 4 (月 10〜12): Pマーク審査申請

1. ⏳ JIPDEC への申請書提出
2. ⏳ 文書審査（1〜2 ヶ月）
3. ⏳ 現地審査（1 日）
4. ⏳ 指摘事項の是正
5. ⏳ 付与（認定）

**想定所要期間**: 申請準備開始から取得まで約 **10〜12 ヶ月**

**想定コスト**: 申請料 + 審査料 + 付与料で初年度 **約 30〜100 万円**（事業規模・業種により変動）。加えて内部作業工数。

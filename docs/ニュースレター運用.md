# ニュースレター運用ガイド

**収集先**: Firebase Firestore `newsletter_subscribers` コレクション
**フォーム**: `js/newsletter-signup.js`（疾患 LP / FAQ / About に自動表示）
**ルール**: `firestore.rules` `match /newsletter_subscribers/{emailHash}`
**作成**: 2026-04-23

## 1 回限りのセットアップ（5 分）

### 1. `/firebase-config.json` を公開ファイルとして配置

Firebase の Web SDK 設定は **公開しても安全** です（セキュリティは Firestore Rules で担保）。
Firebase 公式: <https://firebase.google.com/docs/projects/learn-more#config-files-objects>

リポジトリのルートに `firebase-config.json` を作成:

```json
{
  "apiKey": "AIza...（Web API キー）",
  "authDomain": "your-project.firebaseapp.com",
  "projectId": "your-project",
  "storageBucket": "your-project.appspot.com",
  "messagingSenderId": "1234567890",
  "appId": "1:1234567890:web:abcdef"
}
```

⚠️ <strong>サービスアカウントキー（admin SDK の private key）ではなく、Web SDK 用の公開設定</strong>です。Firebase Console → プロジェクトの設定 → 全般タブ → 「アプリ」セクションの「Firebase SDK snippet (構成)」から取得できます。

### 2. firestore.rules に許可ルールを追加（必須・手作業）

> ⚠️ <strong>重要</strong>: 以下のルールブロックを `firestore.rules` の他の `match` ブロックと同じレベル（`match /databases/{database}/documents` の中）に追加してから、`firebase deploy --only firestore:rules` を実行してください。これがないとフォームは `permission-denied` で必ず失敗します。

```javascript
// 健康日記 ニュースレター登録 — 匿名 create を許可、shape-validated
match /newsletter_subscribers/{emailHash} {
  allow create: if request.resource.data.email is string
                && request.resource.data.email.size() > 4
                && request.resource.data.email.size() <= 200
                && request.resource.data.email.matches('^[^@\\s]+@[^@\\s]+\\.[^@\\s]+$')
                && request.resource.data.subscribedAt is string
                && (!('disease' in request.resource.data)
                    || (request.resource.data.disease is string
                        && request.resource.data.disease.size() <= 50))
                && (!('source' in request.resource.data)
                    || (request.resource.data.source is string
                        && request.resource.data.source.size() <= 200));
  allow update: if request.resource.data.email == resource.data.email
                && request.resource.data.diff(resource.data).affectedKeys()
                   .hasOnly(['subscribedAt', 'consent', 'disease', 'source']);
  allow read:   if isAdmin();
  allow delete: if isAdmin();
}
```

デプロイ:
```bash
firebase deploy --only firestore:rules
```

許可ルール未追加時の挙動: フォームは UI を表示しますが、送信時に「登録に失敗しました: permission-denied」と出ます。本番投入前に必ずルールをデプロイしてください。

### 3. 動作確認

- 任意の疾患 LP（例 `/me-cfs.html`）を開く

- テストメールで登録 → Firebase Console → Firestore → `newsletter_subscribers` コレクションに 1 件追加されていることを確認

## 登録データのスキーマ

```
newsletter_subscribers/{emailHash}
  email: "user@example.com"            # 必須
  subscribedAt: "2026-04-23T10:30:00Z" # ISO 8601
  consent: true                         # 必須（GDPR/個情法）
  disease: "mecfs"                      # 任意（流入元の疾患タグ）
  source: "/me-cfs.html"                # 任意（流入元 URL）
```

emailHash はクライアント側で生成される 8-12 文字の base36 値で、同じメールでの重複登録を自然に upsert します。

## 月次配信のフロー（推奨）

### A. シンプル運用（Mailchimp / ConvertKit 不使用）

1. 月初に Firebase Console から CSV エクスポート
2. ローカルで [Mailtrap](https://mailtrap.io) / [Resend](https://resend.com) / [Postmark](https://postmarkapp.com) を使い BCC で送信
3. もしくは Gmail から手動 BCC（〜100 件まで実用的）

無料運用の限界は <strong>500-1,000 件</strong> です。

### B. メールサービス連携（推奨：1,000 件超えたら）

- **Buttondown**（月 $9〜・100 件まで無料・シンプル UI・推奨）
- **ConvertKit**（月 $15〜・1,000 件まで無料・タグ運用が強力）
- **Mailchimp**（月 $13〜・500 件まで無料・統合が広い）

連携方法:
1. Cloudflare Worker `worker/newsletter-sync.js`（未実装、必要時に作成）が Firestore からエクスポート
2. 各サービスの API へバッチで POST
3. cron で 1 日 1 回 sync

### C. Firebase Extensions「Trigger Email」

Firebase 公式の拡張機能「Trigger Email」を使うと Firestore への書き込みをトリガーに自動メール送信ができます。SendGrid / Mailgun / SMTP のいずれかが必要。

## トラブルシューティング

### 「登録基盤が一時的に利用できません」エラー
→ `/firebase-config.json` がないか不正。前述のセットアップ手順を確認。

### 登録ボタンを押しても何も起こらない
→ ブラウザコンソールで `[newsletter]` のログを確認。Firestore Rules のデプロイ漏れや、CDN からの Firebase SDK ロード失敗が原因のことが多い。

### Firestore に書けない（permission-denied）
→ `firestore.rules` の `newsletter_subscribers` セクションをデプロイしているか確認:
```bash
firebase deploy --only firestore:rules
```

# 健康日記 データ構造リファレンス

最終更新: 2026-04-28

ソースコードを直接読み取って整理したデータ構造の全体図。
`js/*.js` / `firestore.rules` / `worker/*.js` を一次資料として参照すること。

---

## 1. 永続化レイヤー全体図

| レイヤー | 用途 | 容量目安 | コード |
|---------|------|---------|-------|
| **メモリ（Store）** | 全画面で共有する state | 揮発 | `js/store.js` |
| **localStorage** | state の自動永続化（小さい JSON） | ~5MB | `js/store.js` (`cc_*` キー) |
| **IndexedDB** | 写真 dataURL / 大型 JSON / 書き込みキュー | ~50MB+ | `js/idb.js` |
| **Firestore** | クラウド同期（マルチデバイス） | 無制限 | `js/firebase-backend.js` |
| **Cloudflare KV** | PubMed 要約キャッシュ（疾患別） | per-key 25MB | `worker/research-updater.js` |

**保存フロー**:
1. Store の `set()` / `update()` が呼ばれると localStorage に同期保存（`cc_{key}`）
2. 写真など大型データは IndexedDB に分離保存
3. ログイン中ユーザーは Firestore に並行同期（autoSync）
4. オフライン時の Firestore 書き込みは `pendingWrites` キューに保留→復帰時に flush

## 3. IndexedDB

DB 名: `health-diary`、バージョン 1。

| Object Store | Key | レコード構造 | 用途 |
|-------------|-----|------------|-----|
| `images` | `id` | `{id, dataUrl, savedAt: timestamp}` | 写真の本体（localStorage 容量回避） |
| `largeData` | `key` | `{key, data, savedAt: timestamp}` | 任意の大型 JSON（分析履歴等） |
| `pendingWrites` | `id` (auto) | `{collection, entry, createdAt, retries}` | オフライン書き込みキュー |

API:
- `idb.saveImage(id, dataUrl)` / `idb.getImage(id)`
- `idb.saveLargeData(key, data)` / `idb.getLargeData(key)`
- `idb.enqueueWrite(collection, entry)` / `idb.flushPendingWrites()`

`pendingWrites` は `collection` インデックスで検索可能。Firestore 復帰時にコレクション単位で flush。

### 4.2 セキュリティルール（`firestore.rules`）

| パス | Read | Write |
|------|------|-------|
| `/users/{userId}` | uid 一致 OR `agewaller@gmail.com` | uid 一致 |
| `/users/{userId}/{sub}/{doc}` | 同上 | 同上（管理者は read のみ） |
| `/admin/{doc}` | 認証済み全員 | `role == 'admin'` または `agewaller@gmail.com` |
| `/inbox/{hash}/{kind}/{msgId}` | 認証済み AND `resource.data.hash == hash` | Create: `processed==false && kind in ['plaud','data'] && text.size > 0 AND ≤ 200KB`、Update: `processed==true && hash 一致`、Delete: 禁止 |

| `/prompts/`, `/aiModes/`, `/auditLogs/`, `/safetyEvents/` | Admin のみ read | クライアント書き込み禁止 |

## 6. 記録カテゴリ（`DATA_CATEGORIES`）

各記録の共通フィールド:

```javascript
{
  id: string,            // Date.now().toString(36) + Math.random()
  timestamp: ISO string,
  category: string,      // categoryToStateKey マッピング
  ...                    // カテゴリ固有フィールド
}
```

カテゴリ別の主要フィールド:

| カテゴリ ID | 名称 | フィールド | 型・範囲 |
|-----------|------|----------|---------|
| `symptoms` | 今日の体調 | `condition_level` | int 1-10 |
| | | `fatigue_level` / `pain_level` / `brain_fog` / `sleep_quality` | int 0-7 |
| | | `pem_status` | boolean |
| `vitals` | バイタル | `heart_rate` | int (bpm, 40-200) |
| | | `blood_pressure` | string ("120/80") |
| | | `temperature` | float (℃, 34-42, 0.1 刻み) |
| | | `spo2` | int (%, 80-100) |
| `medication` | 服薬 | `medications`, `dosage` | string |
| `sleep` | 睡眠 | `duration` | float (時間, 0-24, 0.5 刻み) |
| `activity` | 活動 | `steps` | int (0-50000) |
| `nutrition` | 食事 | `meals` | string |
| `blood_test` | 検査結果 | `wbc`, `rbc`, `hemoglobin`, `crp` | numeric |
| `mental` | 気分 | `mood` | int 1-10 |
| `photos` | 写真 | `meal_photos`, `test_results_photo` | base64 dataURL |

### Health Score 計算（`store.js:267-291`）

過去 7 日の症状から算出:
- `condition_level` (1-10) があれば `avg * 10` に正規化
- なければ `fatigue_level` / `pain_level` / `brain_fog` 平均で症状スコア + `sleep_quality` でスリープスコア
- 最終: `symptomScore * 0.7 + sleepScore * 0.3`（0-100）

## 8. Cloudflare Worker

### `worker/anthropic-proxy.js` — Claude API プロキシ

**エンドポイント**: `POST /api/anthropic`

**リクエスト**:
```javascript
{
  model: string,
  messages: [{ role: 'user'|'assistant', content: string }],
  system?: string,
  max_tokens?: number,
  temperature?: number
}
```

**レスポンス**:
```javascript
{
  content: [{ type: 'text', text: string }],
  usage: { input_tokens, output_tokens }
}
```

**セキュリティ**:
- CORS: `https://cares.advisers.jp` のみ許可
- `x-api-key` ヘッダー必須（環境変数フォールバック削除）
- サーバー側で拒否検出・サニタイズ・リトライ・安全プロンプト注入
- クライアントは `{disease, symptoms}` と `promptKey` のみ送信可

### `worker/plaud-inbox.js` — メール受信

Cloudflare Email Routing で `plaud-{hash}@inbox.cares.advisers.jp` 受信。
RFC 5322 パーサーで multipart / quoted-printable / base64 を解釈。
`/inbox/{hash}/plaud/{messageId}` に書き込み。

### `worker/professional-mailer.js` — 専門家向けメール送信

**リクエスト**:
```javascript
{
  to: email,
  toName?: string,
  replyTo: email,
  replyToName?: string,
  subject: string,
  text: string,           // ≤ 50KB
  meta?: { programId, programName, applicationId, userUid }
}
```

**プロバイダ**: Resend API（`RESEND_API_KEY` 設定時）→ フォールバック: MailChannels

### `worker/research-updater.js` — PubMed 要約 Cron

毎週日曜 03:00 UTC（日本時間 月曜 12:00）実行:
1. PubMed esearch で疾患別クエリ検索（過去 30 日、最大 10 件）
2. Claude Haiku で要約
3. KV にキャッシュ: `research:{diseaseId}` → `{highlights: [...], updatedAt: ISO}`

`anthropic-proxy` が KV から読んでプロンプトに注入する。

### `worker/learning-orchestrator.js` — 学習コンパニオン

**エンドポイント**:
- `GET /health` — ヘルスチェック
- `GET /me` — 認証ユーザー情報（uid, email, isAdmin）
- `POST /chat` — `{ sessionId, mode, userMessage }`
- `GET/POST /admin/...` — 管理者操作（prompts / ai-modes / audit-logs）

プロンプト・API キー・監査ログはすべて Server-side のみ。

## 10. 重要な制約・注意点

### Store
- localStorage クォータ（~5MB）超過時は toast で警告。写真は IndexedDB へ。
- スキーマ変更時は `Store.MIGRATIONS` にバージョン追加。
- `localStorage.clear()` 禁止 → `store.clearAll()` を使う（Firebase 設定が消える）。

### Firestore
- `dataUrl` / `previewImage` は Firestore に保存しない（容量・コスト）。
- `createdAt` は必ず `serverTimestamp()` を使う。
- Admin（agewaller@gmail.com）は全ユーザーデータを read 可能、write 不可。
- `/inbox/` は Worker のみが create する（クライアントは read のみ）。

### Worker
- Anthropic model id は `MODEL_MAP` 経由で解決（datestamped 名のハードコード禁止）。
- API キーは `admin/config` または `admin/secrets` に保存（コードハードコード禁止）。

### CONFIG
- 疾患カテゴリの追加: `DISEASE_CATEGORIES` に `{id, name, icd}` を足し、必要なら `DEFAULT_PROMPTS` / `DISEASE_EPIDEMIOLOGY` も更新。
- AI モデル変更: `AI_MODELS` を更新後、`MODEL_MAP` も同期。

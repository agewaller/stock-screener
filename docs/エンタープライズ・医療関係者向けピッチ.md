## エグゼクティブサマリー

健康日記は、**慢性疾患患者の症状管理を AI で支援する無料ウェブアプリ**です。日本の慢性疾患患者 約 1,500 万人（ME/CFS / Long COVID / 線維筋痛症 / IBS / 橋本病ほか 80 疾患）を主たるターゲットとし、毎日の症状記録から **主治医提出用カルテ** を自動生成、**14 軸のセルフケア提案**、**17 種の社会保障制度マッチング** までをワンストップで提供します。

## 1. 解決する課題

### 1-1. 慢性疾患は「見えない病」で経済負担が大きい

- 日本の慢性疾患患者総数: **約 5,000 万人**（厚労省・複数疾患重複含む）
- 主要疾患の推計患者数:
  - **ME/CFS**: 10-30 万人（厚労省難治性疾患研究班）
  - **Long COVID（コロナ後遺症）**: 700 万人以上（日本感染症学会推計）
  - **線維筋痛症**: 200 万人（日本線維筋痛症学会）
  - **IBS（過敏性腸症候群）**: 1,200 万人（人口の 10%、日本消化管学会）
  - **橋本病（甲状腺機能低下症）**: 300-500 万人（女性の 10-20% が抗体陽性）
  - **慢性うつ病**: 500 万人
- **経済損失**: 厚労省試算で慢性疾患による年間労働損失は **約 4 兆円**

### 1-2. 患者の「3 分診察」問題

慢性疾患は症状が日々変動し、客観的検査では捉えにくいため、診察 3 分の限られた時間で症状を伝えきれず、医師との情報非対称性が治療効果を阻害しています。

> 米国医学研究所（IOM）2015 年報告書: 「ME/CFS 患者の 80% は適切な診断に至らない」
> NICE Guideline 2021: 「ME/CFS は患者の自己報告データ（症状日記）を診察の一次情報源とすべき」

### 1-3. 既存の症状記録アプリの限界

| アプリ | 課題 |
|---|---|
| Bearable（英国） | 英語、AI 分析なし、CSV 出力のみ |
| Visible（米国） | $15/月、ME/CFS 単独、Polar アームバンド必要 |
| Apple Health | 慢性疾患専用機能なし、医師提出用フォーマットなし |
| Symple | 開発停滞、シンプルな日記のみ |

**日本語ネイティブ × AI 分析 × 主治医提出レポート × 完全無料** という組合せは、健康日記が唯一です。

## 2. プロダクトの特徴と強み

### 2-1. 80+ 慢性疾患を ICD-11 ベースで体系的にカバー

- WHO の国際疾病分類（ICD-11、2022 年発効）に準拠
- 各疾患に専門指標（PEM 頻度、Bristol 便形状、TSH/FT4/FT3、HRV 等）を自動アサイン
- 複数疾患の併発（ME/CFS + Long COVID + 橋本病 等）を 1 画面で追跡
- 公開済の疾患別ガイド（10 ページ）: <https://cares.advisers.jp/me-cfs.html> ほか

### 2-2. AI による「主治医提出用 患者サマリーレポート」自動生成

過去 90 日の症状・服薬・食事・検査結果を、Anthropic Claude（最新 LLM）が読み込み、医療カルテ風 Markdown ドキュメントを **ワンクリック** で生成します。

**含まれるセクション**:
1. 基本情報（疾患・対象期間・記録件数）
2. 主訴 Top 3（強度・頻度・初発時期）
3. 症状経過（時系列サマリー、誘因仮説）
4. 現在の服薬・サプリメント（用量・効果・副作用の表形式）
5. 直近の検査結果（H/L 異常値フラグ付き）
6. 試した対処法とその効果（食事・運動・併用療法）
7. 生活影響度（QOL/ADL の客観指標）
8. 主治医への質問・相談事項（AI が最大 5 項目を候補提示）
9. 付録: 原データ集計

医師の診察効率向上 + 患者の伝達不安解消の両面から QOL 改善に寄与します。

### 2-3. 14 軸ローテーションによる「今日の新しい打ち手」提示

慢性疾患のセルフケアは選択肢が多すぎて、患者は同じ助言の繰り返しに陥ります。健康日記は **14 軸**（食事 / 運動 / 瞑想 / サプリ / 睡眠 / 検査 / 先端医療 / コミュニティ / 専門医 / バイオハック / 東洋医学 / 環境 / 心理 / 社会）から、日付シードで毎日異なる軸を提示。**2 週間で全角度を体験** できる設計です。

### 2-4. 補助金・社会保障制度マッチング

選択疾患・年齢・就労状況に基づき、**17 種の公的制度** から該当候補を自動抽出:

- 傷病手当金（最長 1 年 6 ヶ月、月給の 2/3）
- 障害年金（月 6.5-20 万円）
- 自立支援医療
- 高額療養費制度
- 医療費控除
- 精神障害者保健福祉手帳
- 失業手当（特定理由離職者）
- 生命保険の就業不能給付
- 等

社会保険労務士・税理士への依頼メールを AI が下書きします。慢性疾患患者の経済負担軽減を仕組みで支援する点は、競合製品にない差別化要素です。

### 2-5. 完全プライバシー設計（医療情報基準）

| 基準 | 対応状況 |
|---|---|
| 個人情報保護法（特定個人情報、第 28 条）| 準拠 |
| JIS Q 15001:2017（個人情報保護マネジメントシステム）| 準拠運用、P マーク取得申請中 |
| 厚労省・経産省・総務省 3 省 2 ガイドライン（医療情報取扱事業者）| 準拠宣言を公開 |
| GDPR（EU）| 設計上適合（cookieless 計測、明示的同意） |
| データ保持| Google Cloud 東京リージョンに暗号化保存。**運営者を含む第三者一切アクセス不可** |
| クッキー| 必須機能のみ。広告 / トラッキング Cookie ゼロ |

医療情報を扱う事業として **保険会社・健保組合の DD（デューデリジェンス）に耐える** 設計です。

### 2-6. 完全無料・広告なし・課金なし（個人ユーザー向け）

慢性疾患患者の経済的事情を考慮し、**個人ユーザーは永続的に無料** での提供を約束しています。運営費は代表のコンサルティング業務収益、Amazon アソシエイト収益、将来的な法人プランで賄う設計。

> 「闘病経験から、月額課金を提示するのは残酷だと感じた」（代表・山口）

この方針は、患者会・医療関係者からの推薦においても重要な信頼要素となっています。

## 4. 大企業向けユースケース

### 4-1. 健康経営優良法人 / 健保組合の Wellbeing 施策

#### 課題
- 慢性疾患による **休職・離職の早期発見** が困難
- ストレスチェック（労安法第 66 条の 10）の **補完ツール** が不足
- メンタルヘルス施策の効果測定が難しい

#### 健康日記が提供する解決策
- 従業員が自発的に記録 → 客観的な体調指標が蓄積
- ストレスチェックでは捉えられない **慢性身体症状**（疲労 / 痛み / 認知機能）を可視化
- 産業医ダッシュボード（ロードマップ）で **集計データのみ** を共有（個別データは閲覧不可）

#### 想定 ROI（中規模企業 500 名想定）
- 慢性疾患による休職率 5% × 休職コスト ¥3M/件 = 年 ¥75M の損失
- 早期発見による休職予防効果 30% = 年 **¥22M の損失回避**
- ¥3,000-5,000/従業員/月 × 500 名 = ¥18-30M/年（コストパフォーマンス良好）

### 4-2. ESG / Diversity & Inclusion の推進

- 障害者雇用促進法・改正労働施策総合推進法の文脈で「**慢性疾患を持つ社員のサポート**」は ESG 評価項目

## 5. 医療関係者向けユースケース

### 5-1. クリニック向け患者推奨ツール

#### 想定する診療科
- 神経内科・精神科（ME/CFS、線維筋痛症、Long COVID）
- 内分泌内科（橋本病、甲状腺機能低下症）
- 消化器内科（IBS）
- 心療内科（自律神経失調症、慢性疲労）
- 総合診療科

#### 医師にとっての利点
1. **診察前情報の質向上**: 患者が紙ノートではなく構造化レポートを持参
2. **症状経過の客観把握**: 自由記述だけでなく数値データ（1-10 スコア、頻度）
3. **服薬コンプライアンスの可視化**: 用量変更日・主観的効果が時系列で見える
4. **副作用の早期発見**: AI が異常パターンを検知、患者にも気づきを提供
5. **治療効果の定量評価**: LDN・SSRI・甲状腺ホルモン補充など、効果判定が困難な薬剤の評価支援

#### 患者の主治医への提出フロー（実装済）

1. 患者がアプリで毎日記録（テキスト・写真・PDF）
2. 受診前にワンクリックで患者サマリーレポート生成
3. PDF / Markdown / 印刷 → 診察時に持参 or 電子カルテにペースト
4. 医師は冒頭の「主訴 Top 3」と「主治医への質問」だけ読めば 30 秒で要点把握

### 5-2. 専門医・研究者向け匿名化データ協力（オプトイン）

将来的に、患者の明示的同意のもとで以下を提供予定（2026 年下半期検討）:

- 匿名化された症状時系列データ（Long COVID / ME/CFS 研究用）
- 治療法ごとのアウトカム（PRO ベース）
- 集団レベルでの誘因相関分析

研究機関との共同研究契約により、データ提供条件を厳格に運用します。

### 5-3. 医療機器（SaMD）でないことの明確化

健康日記は **薬機法上の医療機器ではありません**:

- 診断・治療・処方の代替を行わない
- AI 出力はすべて「参考情報」であることを UI で明示
- 「治る」「効く」等の断定表現を使わない（プロンプトでも禁止）
- 主治医への相談を明示的に推奨

医療機関での導入時の規制リスクは限定的です。

## 6. 技術スタックと運用基盤

### 6-1. 技術選定

| 領域 | 技術 | 選定理由 |
|---|---|---|
| フロントエンド | Vanilla JS SPA | フレームワーク依存最小化、長期保守性 |
| 認証 | Firebase Authentication | Google OAuth + Email/Password、無料枠で運用可 |
| データベース | Firestore（東京リージョン） | リアルタイム同期、暗号化、スケーラビリティ |
| AI | Anthropic Claude（最新版） | 長文コンテキスト、日本語精度、医療系の慎重な応答 |
| ホスティング | GitHub Pages + Cloudflare | コスト最小、可用性高 |
| プロキシ | Cloudflare Workers | API キー秘匿、CORS 制御 |
| 解析 | Cloudflare Web Analytics | Cookie 不使用、Pマーク準拠 |

## 1. 本宣言の目的

シェアーズ株式会社（以下「当社」）が提供する「健康日記」（以下「本サービス」）は、慢性疾患を持つ利用者が自身の健康状態を記録・整理し、AI を活用して生活改善のヒントを得ることを目的としたツールです。

本宣言書は、本サービスが 3省2ガイドライン（厚生労働省・経済産業省・総務省が共同で公表する医療情報取扱いに関するガイドライン群）のうち、本サービスに該当する要求事項に準拠することを自主的に宣言するものです。第三者機関による監査を経た認証ではなく、自主的な準拠宣言であることを明示します。

## 3. 本サービスの位置付けとガイドライン適用範囲

本サービスは以下の性質を持ちます:

- **対象データ**: 利用者本人が自発的に入力する健康記録（症状・バイタル・服薬履歴・食事・会話記録等）= **PHR (Personal Health Record)**
- **非対象**: 電子カルテ・診療録・医療機関間の医療情報連携・処方・診断行為
- **提供形態**: 個人向け Web アプリケーション（ブラウザ単独動作 + クラウド同期）

この位置付けより:

- 厚生労働省ガイドラインのうち「医療機関等における義務事項」（電子カルテ管理、診療録保存、医師の法的責任等）は **直接の適用対象ではありません**。
- 経済産業省・総務省ガイドライン「医療情報を取り扱う情報システム・サービスの提供事業者における安全管理ガイドライン」の **事業者としての要求事項には該当**するものとして、自主的に参照・運用します。
- 併せて「個人情報の保護に関する法律」（個人情報保護法）の **要配慮個人情報** に該当する情報を取り扱うため、本宣言はそれらの保護措置も包括します。

## 4. 遵守項目

### 4.1 組織的安全管理措置

| 項目 | 実施状況 |
|---|---|
| 個人情報保護管理責任者の設置 | 代表者（agewaller / info@bluemarl.in）を責任者として指名 |
| 役割・権限の明確化 | 運営は代表者が直接実施。第三者による閲覧・編集権限なし |
| 事故発生時の報告体制 | 72 時間以内通知の内部ルール確立（§5 参照） |
| 定期的なレビュー | 年 1 回以上、本宣言と運用の整合を見直す |
| 内部規程の整備 | プライバシーポリシー、データ管理手順、インシデント対応手順を文書化 |

### 4.2 人的安全管理措置

| 項目 | 実施状況 |
|---|---|
| 個人情報取扱者の限定 | 代表者 1 名に限定。業務委託・アルバイト等で扱わせない |
| 守秘義務 | 代表者は個人情報保護法上の守秘義務を認識し遵守 |
| 教育・啓発 | 個人情報保護法・3省2GL の改訂を定期フォローアップ |
| アクセスログの保持 | Firebase Authentication と Firestore のアクセスログを Google Cloud 標準機能で保持 |

### 4.3 物理的安全管理措置

| 項目 | 実施状況 |
|---|---|
| 物理媒体の非保持 | ユーザーデータは物理媒体（HDD・USB 等）に保存せず、すべて Google Cloud Firestore に保管 |
| 運営者端末の保護 | 代表者端末は端末ロック・暗号化ディスク・画面ロックを常時有効化 |
| データセンター | Google Cloud (asia-northeast1 / 東京リージョン)。入退室管理・ISO 27001 等認証済み |

### 4.4 技術的安全管理措置

| 項目 | 実施状況 |
|---|---|
| 通信経路の暗号化 | ブラウザ ↔ Firestore / AI API の全経路で TLS 1.2 以上を強制 |
| 保存データの暗号化 | Google Cloud 標準の AES-256 相当暗号化を適用 |
| アクセス制御 | Firebase Authentication (Google OAuth / Email-Password) + Firestore Security Rules によるユーザー単位の完全分離。本人以外は読み書き不可 |
| 認証情報の分離 | API キーは localStorage 上に保持、Firestore の secrets サブコレクションは本人のみ閲覧可 |
| 監査ログ | 管理画面で API 使用量・最終アクセス日時を記録・確認 |
| 脆弱性管理 | 依存ライブラリ（Chart.js, Firebase SDK）の CVE を定期確認、速やかに更新 |
| 二要素認証 | Google アカウント経由の OAuth を推奨（Google 側の 2FA が適用される） |
| バックアップ | Firestore は Google Cloud の多重化により自動的に複数リージョンに冗長化される |

## 5. インシデント対応体制

情報セキュリティインシデント（漏洩・滅失・毀損等）を認知した場合、以下を実施します:

1. **即時対応 (認知から 1 時間以内)**: 被害拡大防止のためのアクセス遮断・原因調査開始
2. **利用者通知 (認知から 72 時間以内)**: 影響を受ける利用者の登録メールアドレスに以下を通知
   - インシデントの概要
   - 漏洩した情報の項目
   - 原因と対応状況
   - 再発防止策
   - 利用者が取るべき対応（パスワード変更等）
3. **個人情報保護委員会への報告**: 個人情報保護法第26条に基づき、要配慮個人情報の漏洩が疑われる場合は遅滞なく報告
4. **事後対応**: 再発防止策の実装と公表

**連絡窓口**: info@bluemarl.in

## 6. 委託先管理

本サービスは以下のサブプロセッサ（処理委託先）を利用しています。いずれも API 経由で送信されたデータをモデル学習に使用しないことを公式ポリシーで明示しています。

| 委託先 | 役割 | 所在地 | 認証 |
|---|---|---|---|
| Google Cloud (Firebase) | データ保存・認証・ホスティング | 東京（asia-northeast1） | ISO/IEC 27001, 27017, 27018, 27701, SOC 1/2/3 |
| Cloudflare | Edge Worker / Email Routing | グローバル | ISO/IEC 27001, SOC 2 Type II |
| Anthropic | AI 分析（Claude API） | 米国 | SOC 2 Type II |
| OpenAI | AI 分析（GPT-4o API） | 米国 | SOC 2 Type II |
| Google | AI 分析（Gemini API） | 米国 | ISO/IEC 27001, SOC 2/3 |

### 外国にある第三者への提供に関する補足

Anthropic・OpenAI・Google (Gemini) は米国を本社とする事業者です。個人情報保護法第28条の規定に基づき、本宣言書および本サービスのプライバシーポリシーにおいて以下を情報提供します:

- 提供先の国名: アメリカ合衆国
- 当該国の個人情報保護制度: カリフォルニア州消費者プライバシー法（CCPA）等
- 講ずる措置: 各社の API 利用規約において、送信データのモデル学習への非利用、SOC 2 準拠、TLS 暗号化通信が保証される

利用者は、本サービスの設定画面 → プライバシー設定 → **「AI 送信前の匿名化」** トグルをオンにすることで、個人を特定し得る情報（氏名・連絡先・住所等）を自動マスクしてから AI に渡すことを選択できます。

## 7. 利用者の権利

利用者は以下の権利を行使できます。いずれも追加費用なく利用可能です。

| 権利 | 手続き |
|---|---|
| 開示請求 | 設定画面 → 「すべてのデータをエクスポート」ボタン → JSON 形式で一括ダウンロード |
| 訂正請求 | 各記録画面での編集機能 |
| 削除請求 | 各記録画面での削除機能、または「アカウントを完全削除（退会）」ボタン |
| 利用停止 | ログアウトにより一時停止。恒久的停止は退会ボタン |
| 第三者提供の停止 | 上記 §6 の委託先への送信は、退会により停止 |

- **ガイドライン改訂への追従**: 厚労省・経産省・総務省のガイドライン改訂に合わせて本宣言書も更新

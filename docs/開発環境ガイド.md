## 前提条件

| ツール | バージョン | 確認コマンド |
|--------|-----------|-------------|
| Node.js | v20 以上 | `node -v` |
| npm | v10 以上 | `npm -v` |
| Git | 任意 | `git --version` |

> Node.js のバージョンは `.node-version` で v20 に固定しています。
> nodebrew / nvm / fnm など任意のバージョン管理ツールを使用可能です。

## セットアップ

```bash
# リポジトリをクローン
git clone git@github.com:agewaller/stock-screener.git
cd stock-screener

# 依存パッケージをインストール（devDependencies のみ）
npm install
```

`npm install` で以下が自動的にセットアップされます:

- ローカル HTTP サーバー（http-server）
- ESLint / Prettier（コード品質）
- html-validate（HTML 構文チェック）
- wrangler（Cloudflare Worker ローカル開発）
- firebase-tools（Firestore ルールデプロイ）
- husky（pre-commit フック）

## ローカルサーバー

```bash
npm start
```

- http://localhost:8080 で `index.html` が開きます
- キャッシュ無効（`-c-1`）なので編集後にブラウザをリロードするだけで反映されます
- CORS ヘッダー付きなので外部 API 呼び出しのテストも可能です
- 停止は `Ctrl+C`

### Firebase Auth について

Firebase Authentication（Google ログイン）を localhost で使うには、Firebase Console で `localhost` を承認済みドメインに追加する必要があります:

1. https://console.firebase.google.com/project/care-14c31/authentication/settings
2. 「承認済みドメイン」→「ドメインを追加」→ `localhost` を追加

## テスト

### スモークテスト

```bash
npm test
```

`tests/smoke.test.js` が 20+ カテゴリ・70 件のテストを実行します:

| カテゴリ | 検証内容 |
|---------|---------|
| JS 構文 | index.html のインラインスクリプト + Worker ファイルの構文チェック |
| onclick ハンドラ | すべての `app.X()` 参照が実在するメソッドか |
| レンダラー完全性 | 13 画面レンダラーがすべて定義されているか |
| 認証フロー | signInWithGoogle / signInWithEmail / handleSignedInUser の配線 |
| AI エンジン | MODEL_MAP 使用、Vision 対応、API キーガード |
| プロンプト | PROMPT_HEADER の必須指示、INLINE_PROMPTS のキー存在 |
| Google 翻訳リンク | 廃止された translate.google.com 形式の不使用 |
| 写真アップロード | photoUpload / handleModalPhotoUpload の存在 |
| カレンダー連携 | accessToken 抽出パスの多重化 |
| Plaud | subscribeToInbox / saveTranscript の配線 |
| 栄養ダッシュボード | BMR 計算 / nutritionLog の永続化 |
| データ完全性 | export / import の対称性 |
| Worker CORS | anthropic-version ヘッダーの許可 |
| Firestore ルール | inbox create 権限 |
| アクション推奨 | AI URL 生成禁止、検証済みリンク生成 |
| モバイル安全性 | confirm() / alert() の不使用 |
| 多言語 | 4+ 言語・10 言語の directive 対応 |
| デプロイ設定 | CNAME / pages.yml / deploy-worker.yml |
| セキュリティ | escapeHtml / Firebase apiKey / ADMIN_EMAILS |

テストはすべて Node.js 組み込みモジュールのみで実行されます（外部テストフレームワーク不要）。

### テストの書き方

`tests/smoke.test.js` を直接編集してください。テストは以下のパターンで記述します:

```javascript
// ファイルを読み込んで内容をチェック
const html = fs.readFileSync('index.html', 'utf8');
assert(html.includes('someFunction'), 'someFunction が定義されていること');
```

## コード品質ツール

### ESLint

```bash
npm run lint
```

対象: `worker/*.js`, `tests/*.js`（index.html のインラインスクリプトは対象外）

設定ファイル: `eslint.config.mjs`（ESLint v9 flat config 形式）

- `eslint:recommended` ベース
- Worker ファイル: Cloudflare Worker グローバル（Response, fetch 等）を許可
- テストファイル: Node.js グローバル（require, process 等）を許可
- `console.log` は許可（`no-console: off`）
- 未使用引数は `_` プレフィックスで抑制可能

### Prettier

```bash
npm run format        # 自動修正
npm run format:check  # チェックのみ（CI 向け）
```

対象: `worker/*.js`, `tests/*.js`

設定ファイル: `.prettierrc`

```json
{
  "singleQuote": true,
  "semi": true,
  "tabWidth": 2,
  "trailingComma": "es5",
  "printWidth": 120
}
```

> index.html および疾患別 LP（*.html）は `.prettierignore` で除外しています。
> 960KB の巨大ファイルを毎回フォーマットするのは非現実的なためです。

### HTML バリデーション

```bash
npm run lint:html
```

`html-validate` で index.html + 疾患別 LP 7 枚の基本的な HTML 構文をチェックします。

## pre-commit フック

`npm install` 時に husky が自動セットアップされ、`git commit` 時に以下が実行されます:

1. **`npm run lint`** — ESLint エラーがあればコミット中止
2. **`npm run format:check`** — Prettier 差分があればコミット中止
3. **`npm test`** — スモークテスト失敗でコミット中止

フォーマット差分でコミットが止まった場合:

```bash
npm run format     # 自動修正
git add -A         # 修正をステージング
git commit         # 再コミット
```

> フックを一時的にスキップしたい場合は `git commit --no-verify` ですが、通常は推奨しません。

## Cloudflare Worker ローカル開発

### セットアップ

```bash
# 環境変数テンプレートをコピー
cp .dev.vars.example .dev.vars

# .dev.vars を編集して実際の値を設定
```

`.dev.vars` の内容:

```
# anthropic-proxy 用
ANTHROPIC_API_KEY=sk-ant-xxxxx

# plaud-inbox 用
FIREBASE_PROJECT_ID=care-14c31
FIREBASE_API_KEY=（Firebase Web API キー）
```

### anthropic-proxy のローカル起動

```bash
npm run worker:dev
```

- http://localhost:8787 で Worker が起動します
- index.html の管理パネルでプロキシ URL を `http://localhost:8787` に変更してテストできます

### plaud-inbox のローカル起動

```bash
npm run worker:plaud:dev
```

> plaud-inbox は Email Worker のため、ローカルでのメール受信テストには制限があります。
> ロジックの単体テストには `wrangler dev` + `curl` で POST リクエストを送信してください。

## 開発ワークフロー

### 通常の開発サイクル

```bash
# 1. 作業ブランチに切り替え
git checkout yanoshin_work

# 2. ローカルサーバーを起動
npm start

# 3. index.html を編集 → ブラウザでリロードして確認

# 4. テスト実行
npm test

# 5. コミット（pre-commit フックが自動実行される）
git add index.html
git commit -m "変更内容"

# 6. プッシュ → GitHub Pages 自動デプロイ
git push origin yanoshin_work
```

### Worker の開発サイクル

```bash
# 1. .dev.vars を設定済みであること
# 2. Worker をローカル起動
npm run worker:dev

# 3. worker/anthropic-proxy.js を編集 → 自動リロード

# 4. テスト実行
npm test

# 5. コミット＆プッシュ → deploy-worker.yml が自動デプロイ
```

## トラブルシューティング

### `npm start` でポート 8080 が使用中

```bash
# 使用中のプロセスを確認
lsof -i :8080
# 別ポートで起動
npx http-server -p 3000 -c-1 --cors -o
```

### pre-commit フックが動作しない

```bash
# husky を再初期化
npx husky
```

### ESLint のルールを変更したい

`eslint.config.mjs` を編集してください。ESLint v9 の flat config 形式です。

### Prettier で index.html をフォーマットしたい場合

`.prettierignore` から `index.html` を一時的に削除して実行できますが、960KB のファイルのため時間がかかります。通常は不要です。

### Worker のローカル起動でエラー

```bash
# wrangler にログイン
npx wrangler login

# .dev.vars が存在するか確認
cat .dev.vars
```

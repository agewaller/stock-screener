# 禅トラック（ZenTrack）― 製品詳細

- **一言**: 音声（Plaud等の文字起こし）をメールで受け取り、AI が**意識・注意の状態**を含めて1日を解析する個人向けライフログ分析。
- **リポジトリ**: `zen-track/zentrack`（外部AIからは参照不可のため本書に詳細を集約）
- **対象**: toC（音声デバイス所有者、quantified-self/生産性層、自己管理したい30-40代）／ **収益**: toC 課金
- **状態**: β段階

## 1. ビジョン・顧客価値

日記・記録は続かないが、**「話すだけ」**なら続く。音声文字起こしを送るだけで、1日が健康・仕事・信用・資産・時間のスコアとタイムラインに変換され、明日のアクションが提示される。

**禅トラック固有の価値＝「意識・注意の状態」まで解析する点。** 行動や数値だけでなく、その日あなたの意識・注意がどの次元に向いていたか（`attention_dimensions` 一次元〜五次元）を AI が読み解き、総合スコア（`net_value_score`）として可視化する。「何をしたか」だけでなく「どんな意識で過ごしたか」が分かるのが他にない深さ。
（※自己申告で意識の層を“記録”する cares の「意識の焦点」とは異なり、zentrack は発話内容から意識状態を**自動で解析**する。）

**顧客の痛み:**
- 振り返りたいが記録が続かない
- 健康・仕事・お金をバラバラに気にして全体像が見えない
- Plaud等の音声デバイスを持っているが「録ったきり」

**提供価値:** 入力負荷ゼロ（話すだけ）／**意識・注意の状態まで解析**／多面スコアで全体最適／推移で自己理解／データは本人のもの。

## 2. 詳細な機能

### 入力フロー（書く手間ゼロ）
1. アカウント登録（Google / メール＋パスワード, Firebase Auth）
2. ユニークな `mailAlias`（6文字）を自動発行 → `info+{mailAlias}@zentrack.com`
3. Plaud 等が文字起こしを上記アドレスへメール送信
4. **SendGrid Inbound Parse** が Webhook で受信 → mailAlias でアカウント特定
5. 最新プロンプト＋文字起こしを **OpenAI** に渡し分析 JSON を生成 → `daily_analysis` に保存（受信日を対象日とする）

### AI 日次分析（出力スキーマの要点）
- **5軸スコア（0〜5）**: 健康 / 時間 / 仕事 / 信用(trust) / 資産(assets)
- **健康詳細**: カロリー収支（摂取/消費/基礎代謝/バランス）、コア指標カバレッジ、悪化シグナル、回復アクション
- **時間の使い方**: 価値創造/基盤/維持/浪費の比率、ディープワーク回数（25/50/90分）
- **people/context**: 登場人物・関係温度、状況/空間 top3
- **attention_dimensions**: 一次元〜五次元の注意配分
- **net_value_score**（総合）
- **timeline**: 時刻・出来事・カテゴリ・次元・価値ラベルの配列
- **actions_tomorrow**: 明日やること

### 画面
- `/`（ダッシュボード：日次・推移分析）、`/settings`（メールエイリアス確認）、`/prompts`（プロンプト管理）、`/login`・`/register`
- データは閲覧・削除・書き出しに対応（論理削除フラグ）

## 3. 技術・アーキテクチャ

| 役割 | サービス |
|---|---|
| Frontend | Next.js (React) on Render |
| Backend | Spring Boot (Java 17) on Render |
| DB | Neon (PostgreSQL) |
| 認証 | Firebase Authentication |
| メール受信 | SendGrid Inbound Parse（MX を `mx.sendgrid.net` に向ける） |
| メール送信 | SendGrid (SMTP) |
| AI 分析 | OpenAI API |

主要テーブル: `Account`（firebase_uid, mail_alias, role, …）、`DailyAnalysis`（target_date, raw_transcription, analysis_json）、`Prompt`。

## 4. ターゲット顧客

- **Plaud 等の音声デバイス所有者**（最有力＝デバイス購入者が即ターゲット）
- quantified-self / 生産性・ライフハック層
- 副業・独立で自己管理したい30-40代

## 5. マーケティング戦略（提携レバー狙い）

1. **Plaud との共同プロモ/紹介**を最優先で打診（デバイス所有者が直接の見込み客）
2. **X(旧Twitter)の生産性/自己管理界隈**でオーガニック（スコア画面は拡散しやすいビジュアル）
3. **ガジェット系メディア/YouTuber**へレビュー依頼（製品貸与/アフィリ）
4. β招待リストを構築し、継続率（リテンション）を見てから課金導入

**ポジショニング**: 「話すだけで、1日が見える化される。書く手間ゼロのライフログ。」

## 6. 収益モデル（toC）

- 無料お試し（β）→ 継続者向け月額プラン（価格は検証中）
- 将来：デバイス提携プラン、家族/チーム共有、長期推移レポート等を検討余地

## 7. cares との関係（カニバリではなく補完）

- cares = **健康特化・慢性疾患の医療文脈**。zentrack = **生活全体（健康含む5軸）・健常者の自己管理**。
- 両者は同じ「データ主権ジャーナリング＋AI」の DNA を共有するが、対象ユーザーと文脈が異なる。

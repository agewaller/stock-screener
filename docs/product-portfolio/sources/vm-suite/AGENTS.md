# Project Instructions

## 絶対禁止事項

- **マイグレーションの down / rollback を勝手に実行しない**。データが消失する。必ずユーザーの明示的な許可を得てから実行すること。
- **`DROP TABLE`、`DELETE FROM`、`TRUNCATE`、`git reset --hard` 等の破壊的操作を勝手に実行しない**。
- マイグレーションの修正が必要な場合は、ALTER TABLE の新規マイグレーションを作成する。既存マイグレーションの直接修正は down/up が必要になるため、ユーザーの許可なしに行わない。

## コーディングスタイル

**原則: 読んですぐわかるコードを書く。**

- パッチ的な解決方法ではなく、本質的、根本的な実装を行う
- 最も単純な方法で書く。巧妙なコードより愚直なコードを選ぶ
- 不要な状態を持たない。計算で求まるものに変数を割り当てない
- 標準ライブラリで十分なら外部クレートや特殊な型を使わない
- 名前は具体的に。実態と合わない抽象的な名前をつけない
- 1つの関数に1つの流れ。分岐や中間データ構造を減らす

## その他

- 実装後はcodexにレビューを依頼します

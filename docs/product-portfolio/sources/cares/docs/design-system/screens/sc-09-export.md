# SC-09 エクスポート

FN-INT-05。データポータビリティの実現画面。JSON / CSV / PDF / FHIR を選択して非同期エクスポート、完了後 Signed URL でダウンロード。

## 関連

- FN-INT-05 (エクスポート)
- ADR-0001 (データ主権)、ADR-0013 S-7 (cognition-shifting と無関係だが、データ主権原則の核)
- シーケンス: [`../../basic-design/architecture/sequence-export.md`](../../basic-design/architecture/sequence-export.md)

## 設計メモ

- **非同期処理**: POST /api/export → 202 Accepted + jobId → Worker が Cloud Storage に書き出し → メール通知 / 画面更新
- ダウンロードは Signed URL (5 分有効、`sequence-export.md`)
- 写真は GCS のオブジェクトを含めるかオプション (容量大)
- FHIR R4: Observation / Condition / MedicationStatement / DiagnosticReport
- 「ダウンロード」をタップしたら audit_logs に export を記録
- export_jobs テーブル (schema.md にはまだないので追加検討項目)
- データ主権原則のショーケース画面: vendor lock-in なし

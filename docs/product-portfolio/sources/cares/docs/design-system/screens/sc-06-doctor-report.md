## 設計メモ

- audience タブで 3 種 (doctor / sns / relative) を切り替え、UI 文言とプロンプトを変える
- **「次の 1 アクション」/「質問事項」を必ず含める** (cognition-shifting、ADR-0013 S-7)
- コストキャップ残量を **生成前に表示**、上限近い場合は warn (BR-20)
- PDF はサーバ側で render (puppeteer 想定、または md → pdf ライブラリ)
- FHIR は Observation / Condition / MedicationStatement 等の R4 リソース変換
- doctor_reports / summaries テーブルに保存 (再ダウンロード可能)
- 生成後の編集はテキストエディタ風 (markdown プレビュー併設)
- audit_logs に action=export (format=pdf/fhir) を記録

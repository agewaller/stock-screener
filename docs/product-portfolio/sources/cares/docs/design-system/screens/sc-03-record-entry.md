# SC-03 記録入力 (9 カテゴリ + テキスト)

9 カテゴリのいずれか + テキストエントリ + 写真を入力する画面。カテゴリ選択タブで切り替え。

## ASCII ワイヤ (症状タブの例)

```
+--------------------------------------------------+
|  ← 戻る            記録                  [保存]  |
+--------------------------------------------------+
|                                                  |
|  [症状] [バイタル] [薬] [睡眠] [活動]              |
|        [食事] [検査] [気分] [写真] [メモ]          |
|        ───                                       |
|                                                  |
|  日時:  2026/05/27 [9:00 ▼]                      |
|                                                  |
|  今日の体調はどうですか?                            |
|  1 (とても悪い) ────●─── 10 (とても良い)          |
|                       7                          |
|                                                  |
|  睡眠の質は?                                       |
|  1 ──●──────── 10                                |
|         3                                        |
|                                                  |
|  気になっていること (任意)                          |
|  ┌────────────────────────────────────────┐    |
|  │ 朝から頭が重い。昨日の夜更かしのせいかも │    |
|  │                                          │    |
|  │                                          │    |
|  └────────────────────────────────────────┘    |
|  [自動保存中... 数秒ごと]  ← BR-15 (drafts)       |
|                                                  |
|  +------------------------------------------+  |
|  |              この内容で記録する             |  |
|  +------------------------------------------+  |
|                                                  |
+--------------------------------------------------+
```

## 設計メモ

- **タブで 9 カテゴリ + メモ + 写真 を切替**。タブ自体も大きめのチップ
- スライダーで 1-10 入力 (数値入力より直感的、ペルソナ 65 歳)
- メモは自由記述、入力中は drafts テーブルに数秒ごと autosave (BR-15)
- 「自動保存中…」表示でユーザーに見える (旧仕様 localStorage と心理的に等価)
- 「保存」ボタン押下で本体テーブル INSERT + drafts DELETE (FN-DIARY-02)
- 入力テキストは送信前に escape (BR-08)、サーバ側 Zod でも validate
- text_entries.content は BYTEA + AES-GCM 暗号化
- 写真タブ: Cloud Storage 経由でアップロード (gcs_object_path 保存)
- 編集モード: 既存レコード → 同画面に値プリセット、updated_at 更新 (FN-DIARY-05)

# SC-01 ランディング / ログイン選択

未認証ユーザーが最初に見る画面。**GCP Identity Platform (Firebase Auth)** のログイン手段から選ぶ ([ADR-0014](../../adr/0014-auth-switch-to-identity-platform.md))。

## 設計メモ

- ペルソナ (65 歳) 配慮で **ボタン大きめ・テキスト大きめ**
- ライトテーマ固定 (ADR-0007, BR-09: ダークテーマ追加禁止)
- `confirm()` / `alert()` 不使用、インライン UI のみ (BR-08)
- 「Google で続ける」は最上位 (利用比率最大想定)
- **お試し分析フォーム** = 未ログインで日記を 1 件入力し AI 分析を体験 (FN-TRIAL-01 / ADR-0016)。
  本文はブラウザにもサーバにも保存しない。制限: 1 IP 1 日 3 回 + 全体 3000 円/月 (BR-23)。
  結果表示後にログイン誘導 CTA を出す
- 旧設計の「試しに使う (アカウントなし)」ボタン (= 匿名ログイン FN-AUTH-03、未実装) は
  お試し分析フォームに置き換えた。匿名ログイン自体は将来の別機能として要件は維持 (BR-14)
- 「AI」リテラルは出さない (BR-09) — フォームの文言は「分析」で統一

## 設計メモ

- **/admin/key-status は値を返さない、is_set のみ** (ADR-0006、BR-04、secret_status テーブル)
- ADR-0011 改訂の三段認可: Firebase Custom Claim `admin:true` OR (OWNER_EMAIL AND provider=="password") OR ブレークグラストークンのいずれかで通る
- 鍵設定の更新 UI は「新しい値を設定」フォーム → サーバが Secret Manager API を直接叩く (ブラウザに鍵を一切残さない)
- AI モデル管理 (FN-ADMIN-06): ai_models テーブルの CRUD、is_default 切替、is_enabled 切替
- 通知キャンペーン (FN-ADMIN-07): notifications_config、対象フィルタ・送信予定日時設定
- ユーザー管理: アカウント検索、強制ログアウト、ロール付与 (FN-ADMIN-04) は Firebase Custom Claim を Admin SDK で付与（実装は `apps/api/scripts/grant-admin.ts`）
- データ管理: 利用統計、特定ユーザーのデータ閲覧 (audit_logs に actor=admin で記録、本人にも visible_to_user=TRUE で見える)
- ライトテーマ固定 (BR-09)

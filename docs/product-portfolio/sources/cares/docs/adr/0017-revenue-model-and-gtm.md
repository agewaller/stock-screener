# ADR-0017: 収益モデルと Go-To-Market 戦略

- Status: Accepted
- Date: 2026-06-05
- Deciders: オーナー（agewaller@gmail.com）、Claude（提案）

## Context（背景）

staging 実機テスター（Frances H さん）から、収益モデルとプロモーション戦略についての
質問・提案があった（LINE, 2026-06）。これを機に、これまで未定義だった収益・GTM の
基本方針を決定する。検討の経緯と選択肢の比較は
[`docs/product/revenue-and-gtm-memo.md`](../product/revenue-and-gtm-memo.md) に記録。

判断に効く既存制約:

- **データ主権 5 原則 / ブラウザ非 PII**（[ADR-0001](0001-rearchitecture-from-firebase-to-gcp.md) /
  [ADR-0007](0007-browser-pii-prohibition.md)）: 広告・データ二次利用での収益化は思想的に取りづらい。
- **AI コストは月 1000 円/人キャップ**（[ADR-0013](0013-2026-05-27-operational-supplements.md)）:
  1 ユーザあたりの変動費上限が読め、価格設計の土台になる。
- **対象は慢性疾患当事者**（ME/CFS・Long COVID 等）: 就労困難な層が多く価格感度が高い。
- 既存機能の手応え: 医師提出レポート（#16）・補助金申請支援（#36）＝「書類作成の負担軽減」が
  テスターに刺さっている。

## Decision（決定）

### D-1. 収益の主軸: **B2B2C 主軸 + フリーミアム併走**

- 医療機関・支援団体が利用料を負担し、患者へ提供する **B2B2C** を主軸とする。
- 個人は **無料〜低額**で入れるフリーミアムを併走させる（実質的な支払い主体は患者本人ではない構図）。
- データ主権原則と両立し、後述の医師経由 GTM（D-3）と直結する。

### D-2. データ二次利用(D): **当面は選択肢から除外**

- 同意ベースの匿名統計を研究機関へ提供する収益化は **当面行わない**。
- 将来検討する場合は、**独立した ADR** で同意取得・匿名化・再識別防止を厳密に設計することを必須とする
  （ADR-0001 / ADR-0007 の思想を損なわないことが前提）。

### D-3. GTM: **「医師から患者への推奨」を正式戦略に据える**

- 医師の推薦は患者の信頼度が最も高く、最も確実なアプローチ（テスター実感）。
- これに伴い、**医師提出レポート（#16）を「医師向けの価値」として磨く優先度を上げる**
  （外来効率化＝医師が薦める動機になる）。
- 段階: ①当事者コミュニティ口コミ → ②協力医・専門クリニック開拓 → ③患者会・支援団体連携（B2B2C 契約主体）。
- 留意: 医療広告ガイドライン / 景表法（効果効能の断定・誇大表現を避ける）。本アプリは**医療機器ではない**
  立て付け（診断・治療を行わない、記録と寄り添い支援）を明確にする。

### D-4. コミュニティ拡散のゴーサイン条件: **規約/プライバシー確定まで待つ**

- ME/CFS の LINE グループ等への共有（テスター打診）は、**[#24](https://github.com/agewaller/cares/issues/24)
  の規約/プライバシーポリシーの法務レビュー完了 ＋ AI 月次キャップ運用の確認**を満たすまで待つ。
- それまでは現テスターの範囲で継続。雛形のまま外部拡散しない。

## Consequences（結果・影響）

- **優先度の変化**: #16（医師レポート）を医師目線で磨く作業の優先度が上がる。#24（規約/プライバシー）の
  法務レビューが「拡散の前提条件」として明確化された。
- **やらないことの明文化**: データ二次利用での収益化は当面棚上げ（思想リスク回避）。
- **B2B2C 準備**: 医療機関向けの価値提案資料（1 枚）と、医師レポートの医師レビューが次の準備物。

## 未決事項（follow-up）

- **無料/有料の具体的な線引き**（例: 記帳・即レスは無料、エクスポート・深堀分析・補助金支援を有料 等）は
  本 ADR では確定しない。プロダクト成熟と B2B2C 契約形態の具体化に合わせて別途決定する。
- 医療機関向けの契約・課金形態（サブスク単価、施設単位/患者数単位 等）。

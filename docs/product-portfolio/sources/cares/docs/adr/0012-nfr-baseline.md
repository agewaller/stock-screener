# ADR-0012: 非機能要件のベースライン

- Status: Accepted
- Date: 2026-05-26
- Deciders: オーナー、Claude（提案）

## Context

機能要件は ADR-0001〜0011 で網羅。非機能要件（SLA、パフォーマンス、データ保持、バックアップ、DR、セキュリティ、コンプライアンス）も MVP のベースラインを確定する必要がある。

数十人規模 MVP の段階なので、エンタープライズグレードの SLA は不要。ただし医療データを扱う以上、最低限のセキュリティ・データ保護は譲れない。

## Decision

### SLA

**明示的 SLA なし、ベストエフォート**。

- MVP / 数十人規模では SLA を約束しない
- ただし内部目標として可用性は監視（Cloud Monitoring）

### データ保持

- **アクティブデータ**: ユーザーが削除しない限り **無期限**
- **削除ポリシー**: ソフト削除 → 30 日後にハード削除（ADR-0007 の確定事項を継承）
- ハード削除後はバックアップからの復元もポリシー的に拒否

### バックアップ

- **Cloud SQL 自動バックアップ + Point-in-Time Recovery（PITR）**
- **保持期間 30 日**
- 東京リージョン内で冗長化

### DR（災害復旧）

- 東京リージョン内の冗長性で対応
- マルチリージョン DR は MVP スコープ外
- DR テストは半年に 1 回程度、PITR からのリストア演習

### パフォーマンス目標（暫定）

| 項目 | 目標 |
|---|---|
| 非 AI API レイテンシ | p95 < 500ms |
| AI 応答ストリーミング開始 | < 2 秒（以降は LLM 依存） |
| ページロード（LCP） | < 2.5 秒 |
| 想定ストレージ | ユーザー 1 人あたり 100MB / 月（写真込み） |

### セキュリティ

- **保管時暗号化**: Cloud SQL / Cloud Storage は Google 管理鍵で自動暗号化（MVP は CMEK 不要）
- **転送時暗号化**: TLS 1.2 以上必須、HSTS 有効
- **シークレット**: Secret Manager（ADR-0005, 0006）
- **ブラウザ PII 禁止**: ADR-0007
- **監査ログ**: Cloud Audit Logs + アプリ層ログ、ユーザーにも一部公開

### コンプライアンス

- **日本の個人情報保護法を遵守**
- 同意取得 UI、開示請求対応、削除請求対応を MVP に実装
- GDPR / HIPAA 相当は将来必要時に対応（スキーマは GDPR を意識した設計を ADR-0008 で示唆）

### 負荷想定（MVP）

- 数十人規模なら **Cloud Run 1 インスタンス / Cloud SQL db-f1-micro 〜 db-g1-small** で十分
- スケール時に随時拡張

## Consequences

### 良い面

- 明示的 SLA を約束しないことで、運用負担とコストを抑制
- 個人情報保護法ベースの最低限のコンプライアンスは確保
- パフォーマンス目標は監視ベースで継続改善できる
- バックアップ 30 日 + ソフト削除 30 日のダブル保護

### 悪い面

- 法人展開（B2B）時には SLA を約束する必要が出る可能性（再検討）
- マルチリージョン DR がないため、東京リージョン全域障害には弱い
- パフォーマンス目標は暫定。実測で見直しが必要

### 派生タスク

- Cloud Monitoring のダッシュボード・アラート設計
- DR 演習手順書（半年に 1 回の PITR リストア）
- プライバシーポリシー / 利用規約の作成
- 同意取得 UI の実装（フェーズ 5）
- 個人情報開示請求 / 削除請求の運用フロー策定

## Alternatives Considered

- **99.9% SLA を約束**: db HA + Cloud Run マルチリージョンが必要、コスト上がる。数十人規模では過剰。
- **マルチリージョン DR**: 東京リージョン全域障害は確率低。コストに見合わない。
- **GDPR / HIPAA 即対応**: 海外展開・医療機関導入が確定するまで不要。早すぎる最適化。
- **データ保持に明示的上限を設定（例: 10 年）**: ユーザーの「ずっと記録できる」という期待を裏切る。医療データは生涯記録する価値がある。

# 料金設計ドラフト（無料 / 有料ライン + B2B2C）

> ステータス: **ドラフト / 意思決定前**。本書は [ADR-0017](../adr/0017-revenue-model-and-gtm.md) が
> follow-up として明示的に先送りした「無料/有料の具体的な線引き」を詰めるための提案。
> オーナー（agewaller@gmail.com）の決定後、確定事項は ADR-0017 を改訂または新規 ADR 化する。

## 1. 目的

ADR-0017 で決めた **B2B2C 主軸 + フリーミアム併走** を、実際の機能割り当て・価格帯まで具体化する。
ゴールは「①無料でも価値が伝わり、②有料/契約に自然に繋がり、③AI 変動費が破綻しない」線引き。

## 2. 制約・前提（設計の土台）

- **AI 変動費の上限**: 月 **1000 円/人** キャップ（[ADR-0013](../adr/0013-2026-05-27-operational-supplements.md) S-7、
  超過は 422 拒否）。**1 ユーザの最悪原価が読める**＝ここが価格の床。
- **対象**: 慢性疾患当事者（ME/CFS・Long COVID 等）。就労困難層が多く**価格感度が高い**。
- **データ主権**: 広告・データ二次利用での補填は不可（[ADR-0001](../adr/0001-rearchitecture-from-firebase-to-gcp.md) /
  [ADR-0007](../adr/0007-browser-pii-prohibition.md)、ADR-0017 D-2）。**課金がほぼ唯一の原資**。
- **既存の価値資産**: 記帳8カテゴリ、AI（日次/深堀/レポート/研究）、医師提出 PDF（#16）、
  補助金申請支援（#36）、エクスポート（CSV/FHIR/PDF）、お試し AI（[ADR-0016](../adr/0016-public-trial-ai-endpoint.md)）。

## 3. 設計原則

1. **記録は常に無料**。「書く」ことに金銭の壁を置かない（健康日記の本質 + データ蓄積がリテンションの源泉）。
2. **AI と書類作成（医師提出/補助金）を有料価値に寄せる**。原価（AI）と提供価値が高いものを有料側へ。
3. **無料枠の AI は原価が発生する → 必ず量で上限を設ける**（下記 §5）。青天井の無料 AI は禁止。
4. **支払い主体は患者本人とは限らない**（B2B2C）。個人課金は「払える人/払いたい人」の任意。
5. **ユーザーに金額（円）は見せない**（オーナー判断 / [#66](https://github.com/agewaller/cares/issues/66)）。
   「今月いくら使った/上限いくら」をエンドユーザーに表示するのは印象が良くないため撤去する。
   サーバ側の月次キャップ＋422 拒否（ADR-0013 S-7）は**維持**し、上限到達は金額抜きの文言で案内する。
   管理画面の集計 ¥ 表示は運用上残す。

## 4. 無料 / 有料ラインの提案

| 機能 | 無料 (Free) | 個人有料 (Plus) | B2B2C (機関提供) |
|---|---|---|---|
| 記録 8 カテゴリ・編集削除復元 | ✅ | ✅ | ✅ |
| 自由記入＋自動仕分け（#33 S2） | ✅ | ✅ | ✅ |
| 監査ログ閲覧 | ✅ | ✅ | ✅ |
| お試し AI 分析（ログイン前, ADR-0016） | ✅（日次上限） | ✅ | ✅ |
| **AI 日次コメント** | ✅ **1日1回まで** | ✅ キャップ内で随時 | ✅ キャップ内 |
| **AI 深堀 / レポート / 研究** | ❌ | ✅ | ✅ |
| **医師提出 PDF（#16）** | △ 月1回まで | ✅ 無制限 | ✅ 無制限 |
| **補助金申請支援（#36）** | ❌ | ✅ | ✅ |
| エクスポート CSV / FHIR / PDF | ✅ JSON/CSV | ✅ 全形式 | ✅ 全形式 |
| 容量・写真 | 基本枠 | 拡張 | 拡張 |

> △/上限値は仮。実コストと利用実態を見て調整する前提。

## 5. AI コストキャップとの整合（最重要）

無料枠でも AI を出すと原価が発生するため、**無料 = 低頻度の軽い AI のみ**に絞る:

- **Free**: AI は「日次コメント 1日1回（Haiku 級・短文）」＋お試し分析のみ。深堀/レポート/研究は不可。
  → 1 人あたり月原価を **数十円オーダー**に抑え、無料ユーザが増えても破綻しない。
- **Plus / B2B2C**: フル AI を**月 1000 円キャップ内**で開放。原価上限が 1000 円/人なので、
  価格は「1000 円（原価）＋運用・粗利」を上回る水準に設定すれば赤字にならない。
- キャップ到達時は既存どおり 422 拒否（青天井課金事故を防ぐ）。**無料枠にも別途の日次/月次クォータ**を必須実装。

## 6. 価格帯の仮説（要検証・未確定）

- **個人 Plus**: AI 原価上限 1000 円/月を踏まえ、**月 1,000〜1,500 円**前後が損益分岐の目安。
  ただし対象層の価格感度を考えると強気にしづらい → **B2B2C を主軸に置く根拠**（ADR-0017 D-1）。
- **B2B2C**: 医療機関/患者会/支援団体が契約主体。課金単位は「施設サブスク」か「患者数従量」。
  単価は AI 原価（1000 円/人）＋運用を回収できる水準。具体額は商談で詰める。

## 0. 前提（既存の制約から効いてくるもの）

- **データ主権 5 原則 / ブラウザ非 PII**（ADR-0001 / ADR-0007）: 広告・データ二次利用による収益化は思想的に取りづらい。**医療 PII を売らない**前提で設計する。
- **AI コストは月 1000 円/人キャップ**（ADR-0013）: 1 ユーザあたりの変動費上限が読める＝サブスク価格設計の土台になる。
- **対象は慢性疾患当事者**（ME/CFS・Long COVID 等）: 多くが就労困難・経済的に厳しい層。**価格感度が高い**。
- 既存機能の方向性: 医師提出レポート（#16）、補助金申請支援（#36）＝**「書類作成の負担軽減」が刺さっている**（テスター実証）。

## 1. 収益モデルの選択肢

| 案 | 概要 | 相性 | 主なリスク |
|---|---|---|---|
| A. フリーミアム（B2C サブスク） | 記帳・基本 AI は無料、エクスポート/深堀分析/補助金支援等を有料 | データ主権と両立。価格感度が課題 | 当事者層の支払い力が弱い／無料圏で十分になりがち |
| B. B2B2C（医療機関・支援団体が契約） | クリニック/患者会が利用料を負担し、患者へ提供 | **医師推奨 GTM（§2）と直結**。支払い主体が患者でない | 医療機関の導入ハードル・営業コスト |
| C. 助成金 / 寄付 / 公的補助 | 患者会・研究・行政の補助で運営費を賄う | 非営利的で当事者の信頼を得やすい | 継続性・スケールしにくい |
| D. データ二次利用（研究連携） | 同意ベースの匿名統計を研究機関へ | 学術価値は高い | **ADR-0001/0007 と緊張**。慎重な同意設計必須 |

### 初期の推し（叩き台）
- **B（B2B2C）を主軸 + A（フリーミアム）を併走**。
  - 個人は無料〜低額で入り、**実質的な支払いはクリニック/支援団体**（§2 の医師推奨と一体）。
  - D は当面**やらない**（思想リスク）。やるなら独立 ADR で同意・匿名化を厳密設計。

## 2. Go-To-Market（プロモーション）

テスター提案「**医師から患者への推奨**が最も信頼され確実」は、本プロダクトの強みと一致する。

### なぜ医師経由が効くか
- 患者は医師の推薦を強く信頼する（テスターの実感）。
- 既存の**医師提出レポート（#16）**が医師側にも価値（外来効率化）＝**医師が薦める動機**になる。
- 補助金支援（#36）も医療ソーシャルワーカー/支援者の業務を軽くする。

### GTM 仮説（段階）
1. **当事者コミュニティでの口コミ**（現在地）: テスター経由で ME/CFS の LINE グループ等。※拡散前に「規約/プライバシーの確定」「AI キャップ」を整える（Haruka さんの共有打診への回答に直結）。
2. **協力医・専門クリニックの開拓**: PEM/Long COVID を診る医師に、レポート機能を入口に紹介。「医師→患者」推奨ルートを作る。
3. **患者会 / 支援団体との連携**: 補助金申請支援を軸に団体単位で導入（B2B2C の契約主体候補）。

### 留意（医療広告・倫理）
- 「医師の推奨」を打ち出す際は**医療広告ガイドライン / 景表法**に注意（効果効能の断定・誇大表現を避ける）。
- 本アプリは**医療機器ではない**立て付けを明確に（診断・治療を行わない、記録と寄り添い支援）。AI 文言は BR-09（「AI」リテラル非表示）と医療免責を両立。

## 3. 当面の意思決定ポイント（オーナー向け）

- [ ] 収益の**主軸**を A / B / C のどれに置くか（推し: B + A 併走）
- [ ] **D（データ二次利用）を選択肢に残すか**（残すなら独立 ADR で厳格設計、残さないなら明文で除外）
- [ ] **無料/有料の線引き**（例: 記帳・即レスは無料、エクスポート・深堀・補助金支援を有料 など）
- [ ] **医師経由 GTM を正式戦略にするか**（するなら #16 を「医師向け価値」として磨く優先度を上げる）
- [ ] コミュニティ拡散の**ゴーサイン条件**（規約/プライバシー確定 ＝ #24 の法務レビュー、AI キャップ運用、サポート体制）

## 4. 次アクション候補

- 方針が固まったら **ADR 化**（例: 「ADR-00xx 収益モデルと GTM」）。
- B2B2C を採るなら、**医療機関向けの価値提案 1 枚**と、医師レポート（#16）の医師目線レビューを先行。
- 拡散判断のため **#24（規約/プライバシー）の法務レビュー**を先に通す。

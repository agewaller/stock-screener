# cares 簡易マニュアル (ユーザ向け)

cares は **慢性疾患を抱える方の体調記録 + AI による寄り添いふりかえり** を提供する Web アプリです。本マニュアルは利用者がすぐ使い始められるための手順を、画面イメージ付きで説明します。

> 開発・運用面の詳細は [OPERATIONS.md](OPERATIONS.md) を参照してください。

## 1. はじめてのログイン

ブラウザで <http://localhost:3000/> (本番では <https://cares.advisers.jp/>) を開きます。

```
+-----------------------------------------+
|                  cares                  |
|         慢性疾患のための                 |
|         体調記録・情報整理               |
|                                         |
|  [ G  Google でログイン         ]       |
|                                         |
|  ─────────── または ───────────          |
|                                         |
|  Email     [____________________]       |
|  Password  [____________________]       |
|  [ ログイン                     ]       |
|                                         |
|  ▸ パスワードを忘れた場合               |
|  ※ 新規アカウント作成は管理者に連絡    |
+-----------------------------------------+
```

- **Google でログイン**: 普段使っている Google アカウントで OK。1 クリックで開始
- **Email + Password**: 管理者にアカウント作成を依頼してください (個人情報の管理が厳格な運用方針のため、Email+Password の自己 sign-up は無効化されています)
- **パスワードを忘れた場合**: メールアドレスを入力 → 「パスワードを忘れた場合」をクリック → リセットメールが届きます

## 3. 体調を記録する (10 カテゴリ)

ホームの「＋ 今日の体調を記録する」ボタンを押すと、`/record/symptom` に飛びます。他のカテゴリは URL を直接開きます (将来のリリースでカテゴリ選択 UI を追加予定):

| カテゴリ | URL | 内容 |
|---|---|---|
| 症状 | `/record/symptom` | 体調スコア (1-10)、睡眠の質、メモ |
| バイタル | `/record/vital` | 心拍 / 血圧 / 体温 / SpO2 |
| 服薬 | `/record/medication` | 薬名、用量、飲んだ時刻 |
| 睡眠 | `/record/sleep` | 時間、質 |
| 活動 | `/record/activity` | 歩数、距離、アクティブ分 |
| 食事 | `/record/meal` | 食事内容のテキスト |
| 検査 | `/record/blood_test` | WBC / RBC / Hb / CRP など |
| 気分 | `/record/mood` | 気分スコア + タグ |
| 写真 | `/record/photo` | 写真アップ + 自動分類 (§4) |
| メモ | `/record/text` | 自由記述 |

## 4. 写真を撮って AI に分類してもらう

ホームの「📷 写真を追加」をタップすると `/record/photo` に飛びます。

```
+-----------------------------------------+
| 📷 写真を記録する                       |
| 食事 / 薬 / 検査結果 / 処方箋 などの    |
| 写真をアップロードすると、              |
| 自動で分類し、近い時間の記録と          |
| 紐付けます。                            |
|                                         |
| ┌─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐    |
| │              📷                │    |
| │   タップして撮影 / 画像を選択   │    |
| │   JPEG / PNG / WebP、8 MB まで │    |
| └─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘    |
+-----------------------------------------+
```

1. **タップ**して写真を撮影 (スマホは camera が起動) または ファイル選択
2. プレビュー → 自動圧縮 (長辺 1600px、JPEG)
3. アップロード進捗バー → 完了
4. AI が分析中 (最大 30 秒)
5. 結果表示
   ```
   分類: 食事
   ▸ 詳細を見る (展開して栄養情報など)
   ```
6. 同じ時刻 (±1h) に食事記録があれば自動的に紐付き、タイムラインに `📷 食事(写真)` が表示されます

**プライバシー**: 写真は GCP Cloud Storage に暗号化保管。ブラウザに base64 / 画像データを残しません ([ADR-0007](adr/0007-browser-pii-prohibition.md))。表示時は 5 分有効な署名 URL を都度取得してメモリに展開、タブを閉じるとブラウザから消えます。

## 5. AI のふりかえり 3 種

| 種類 | 場所 | 内容 |
|---|---|---|
| 今日のふりかえり | タイムラインのカード | 直近 7 日の症状を 3〜5 文で要約、cognition-shifting な気づき |
| 深堀ふりかえり | タイムラインのカード | 1 日 1 回、根本要因 (Root Cause) に迫る分析。生活・栄養・睡眠・メンタル・環境・服薬を 5 層で評価 |
| 最新研究スキャン | `/research` | 設定で選んだ疾患に基づく最新研究の要点。`{疾患}_research` プロンプト経由 |

**月次上限**: 1 ヶ月あたり **1000 円** までの AI 利用上限があります。超えると「今月の上限に達しました」と表示され、来月 1 日にリセット (BR-20)。

## 6. レポートを作る (医師・SNS・家族向け)

`/report` → 「+ 新規レポート」:

```
+-----------------------------------------+
| 新規レポート                            |
|                                         |
| 対象期間: [ 30 日 ▼]                    |
| 渡す相手: [ 医師 ▼]                     |
|          ・医師 (カルテ風)              |
|          ・SNS  (個人情報伏せ)          |
|          ・家族 (平易な手紙)            |
|                                         |
| [ レポートを生成する          ]         |
+-----------------------------------------+
```

生成された Markdown レポートを画面で読めます。Word やメールにコピペして印刷・共有できます。

## 8. エクスポート / ゴミ箱 / 監査ログ

| 機能 | URL | 説明 |
|---|---|---|
| エクスポート | `/export` | 全データを JSON でダウンロード |
| ゴミ箱 | `/trash` | 削除済み記録を 30 日以内なら復元可 |
| 監査ログ | `/audit-log` | 自分のデータがいつ誰に read/write/delete されたかの履歴 |

## 9. ログアウト

右上のメニュー (`≡`) → 「ログアウト」。ブラウザの IndexedDB と HttpOnly cookie が両方クリアされます。

## 10. トラブル

| 症状 | 原因 / 対処 |
|---|---|
| ログイン後の画面で 401 ばかり出る | Firebase ID token の 1h 有効期限切れ。ページをリロードまたはログアウト → 再ログイン |
| 写真アップロード後に分類が出ない (30 秒待ってもダメ) | 月次 AI 上限到達、または Anthropic API 一時障害。タイムラインから後でリトライ可 |
| 「アカウント作成は管理者に連絡」と言われる | 設計上 email/password 自己 sign-up は無効。管理者に依頼 (Google ログインは自動 provision) |
| パスワードリセットメールが届かない | スパム箱を確認。それでもなければ管理者に再リセットを依頼 |

### 1.1 ミッション

慢性疾患と毎日向き合う当事者に、**体調記録** と **AI による情報整理** を通じて、自己管理の質を高め、寛解への道筋を可視化する。

### 1.2 ポジショニング

- **SaMD（医療機器プログラム）ではない** — 医療行為は行わない
- 「参考情報の提供」と「日記記録の整理」に限定
- すべての出力に「主治医とご相談ください」のディスクレーマーを付与

### 1.3 ターゲットユーザー（ペルソナ）

**メインペルソナ: 65歳女性**

| 属性 | 詳細 |
|------|------|

| IT リテラシー | LINE は使える。ブラウザの翻訳機能は知らない |

| 健康状態 | 慢性疾患を 1 つ以上抱えている |

### 2.2 疫学データ

各疾患には世界患者数・国内患者数・信頼度（Tier 0〜3）・管理密度（high/medium/low）が付与されている。詳細は [docs/世界患者数一覧.md](世界患者数一覧.md) を参照。

### 2.3 疾患別モニタリング指標

以下の 9 疾患 + 共通について、定点観測すべき指標（importance: critical/high/medium）が定義されている。

**ME/CFS**: PEM 発生頻度 / 活動可能時間 / 疲労度 / 睡眠の質 / ブレインフォグ / 疼痛レベル / HRV / NK 細胞活性 / CRP

**うつ病**: PHQ-9 スコア / 気分 / 睡眠時間 / 外出・活動回数 / 人との接触 / 食欲 / 集中力

**線維筋痛症**: 疼痛部位数 / 疼痛強度 / 睡眠の質 / 疲労度 / 朝のこわばり

**Long COVID**: PEM 発生頻度 / 疲労度 / ブレインフォグ / 呼吸困難度 / 安静時心拍数 / SpO2 / D-ダイマー

**POTS**: 臥位心拍数 / 立位心拍数 / 収縮期血圧 / 水分摂取量 / 塩分摂取 / 失神・ふらつき

**2 型糖尿病**: HbA1c / 空腹時血糖 / 体重 / 運動時間 / 血圧

**甲状腺疾患**: TSH / FT4 / FT3 / 疲労度 / 体重 / 抗 TPO 抗体

**IBS**: 腹痛頻度 / 便の形状(Bristol) / 膨満感 / 食事トリガー / ストレスレベル

**不眠症**: 入眠潜時 / 総睡眠時間 / 中途覚醒回数 / 睡眠効率 / 日中の眠気

**共通（全疾患）**: 体調（総合） / エネルギーレベル / ストレスレベル / 睡眠の質 / 気分

### 5.2 プロンプト体系

3 層構造で全 AI 出力を制御する。

#### PROMPT_HEADER（全プロンプト共通ヘッダー）

すべてのプロンプトに注入される基盤指示。以下を定義:

- **役割定義**: 教育的ヘルスインフォメーション・コンパニオン AI
- **安全基準**: SaMD 非該当、医療行為否定、ディスクレーマー必須
- **応答の 5 原則**: ① 寄り添い・共感 ② 希望の言葉 ③ 毎回 SOMETHING NEW（新処方） ④ プロアクティブ提案 ⑤ 長期視点
- **表現ルール**: 専門用語は平易に説明、やさしい口調、箇条書き多用
- **情報量ルール**: 出し惜しみせず、理由と背景も丁寧に、長くても構造化
- **最先端医療知見**: 免疫療法/神経科学/代謝/腸脳軸/バイオハック/先端検査
- **商品推奨基準**: 第三者認証、吸収率、推奨メーカーリスト
- **地域別イベントプラットフォーム**: 日本/米国/英国/EU/豪州/東アジア/東南アジア（10 地域）
- **会話データ分析**: 発話パターン、ストレス 5 段階評価、精神状態推定
- **OODA ループ**: Observe→Orient→Decide→Act の思考フレームワーク

## 12. UI/UX 設計原則

メインペルソナ（65 歳女性）に最適化した 10 原則:

1. **文字は大きく** — 最小 12px、本文 14px 以上
2. **ボタンは大きく** — タップ領域 44px 以上
3. **操作は 1 タップ** — 長押し・ドラッグ不要
4. **専門用語を避ける** — 「SaMD」は使わない、「PEM」→「活動後の倦怠」
5. **色のコントラスト** — 薄いグレーの文字を避ける
6. **リンクは自動翻訳** — ユーザーの言語でページが開く（`translate.goog` 形式）
7. **エラーは優しく** — 技術用語なし、「もう一度お試しください」
8. **初回は手取り足取り** — 3 ステップオンボーディング
9. **不安を消す** — ディスクレーマーは必要だが脅さない表現
10. **継続の動機** — 小さな改善を可視化し、希望を持たせる（ストリーク・バッジ）
